1、ACID      // 事务

    A   原子性     任何改变都是原子性的
    C   一致性     数据库状态一直是一致性的
    I   隔离性     即使交易并发执行，看起来也是串行的
    D   持久性     一旦交易提交了就不可回滚


2、CAP       // 分布式 系统中三者只能满足其二

    C       一致性
    A       可用性
    P       分区容错性           // 选择分布式  ->  就一定要分区

    -----------------------------------------------------------------
    CP
        Zookeeper、Consul

        强一致性    ==>  牺牲可用性      ->  分区同步时，暂停服务

    AP
        Eureka

        可用性     ==>  牺牲一致性      ->  分区同步时，数据短暂不一致
