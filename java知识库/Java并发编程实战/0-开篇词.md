## 0. 开篇词


#### 0.1 你为什么需要学习并发编程？

```


    章法--理论模型：
    
    
        本质：
        
            管程模型：
            
                Java ⾥ synchronized、wait()/notify() 相关的知识很琐碎，看懂难，会⽤更难。
                
                但实际上 synchronized、wait()、notify() 不过是 操作系统领域 ⾥ 管程模型 的⼀种实现⽽已，
                
                Java SDK 并发包⾥的 条件变量 Condition 也是管程⾥的概念，
        
            
            synchronized、wait()/notify()、条件变量这些知识如果单独理解，⾃然是管中窥豹。
            
            但是如果站在管程这个理论模型的⾼度，你就会发现这些知识原来这么简单，同时⽤起来也就得⼼应⼿了。
        
        
        
        管程 与 信号量：
        
            管程 作为⼀种 解决并发问题 的模型：
            
                是继信号量模型之后的⼀项重⼤创新
            
                它与信号量在逻辑上是等价的（可以⽤管程实现信号量，也可以⽤信号量实现管程）
                
                但是相⽐之下管程更易⽤
            
            
            ⽽且，很多编程语⾔都⽀持管程，搞懂管程，对学习其他很多语⾔的并发编程有很⼤帮助
            
            然⽽，很多⼈急于学习 Java 并发编程技术，却忽略了技术背后的理论和模型，⽽理论和模型却往往⽐具体的技术更为重要
        
        
        
        Java SDK：
        
            此外，Java 经过这些年的发展，Java SDK 并发包提供了⾮常丰富的功能
            
            对于初学者来说可谓是眼花缭乱，好多⼈觉得⽆从下⼿
            
            但是，Java SDK 并发包乃是并发⼤师 Doug Lea 出品，堪称经典，它内部⼀定是有章可循的
        
        
    

    
    其实 并发编程 可以总结为三个核⼼问题：
    
        分⼯、同步、互斥
    
    
        分⼯：
        
            如何⾼效地 拆解任务 并 分配给线程
            
        同步：
        
            线程之间 如何 协作
            
        互斥：
        
            保证 同⼀时刻 只允许 ⼀个线程 访问 共享资源
        
        
        
            
    Java SDK：
    
        Java SDK 并发包很⼤部分内容都是按照这三个维度组织的：
        
            Fork/Join 框架就是⼀种 分⼯模式，CountDownLatch 就是⼀种典型的 同步⽅式，可重⼊锁 则是⼀种 互斥⼿段
            
            
        当把并发编程核⼼的问题搞清楚，再回过头来看 Java SDK 并发包：
        
            你会感觉豁然开朗，它不过是针对并发问题开发出来的⼯具⽽已，
            
            此时的 SDK 并发包可以任你“盘”了
            
            
        ⽽且，这三个核⼼问题是跨语⾔的：
        
            你如果要学习其他语⾔的并发编程类库，完全可以顺着这三个问题按图索骥
            
            
        Java SDK 并发包 其余的⼀部分：
            
            并发容器 和 原⼦类
            
            属于辅助⼯具，其他语⾔⾥基本都能找到对应的
        
    
    


    
    并发编程难学吗？
    
        ⾸先，难是肯定的。
        
            因为这其中涉及：
            
                操作系统、CPU、内存 等等多⽅⾯的知识
            
            如果你缺少某⼀块，那理解起来⾃然困难。
        
        
        其次：
        
            很多⼈在学习并发编程的时候，总是喜欢从点出发，希望能从点⾥找到规律或者本质，最后却把⾃⼰绕晕了。
            
            并发编程并不是 Java 特有的语⾔特性，它是⼀个通⽤且早已成熟的领域。
            
            Java 只是根据⾃身情况做了实现罢了，
            
            当你理解或学习并发编程的时候，如果能够站在较⾼层⾯，系统且有体系地思考问题，那就会容易很多。
        
        
        所以，我希望这个专栏更多地谈及：
        
            问题背后的本质、问题的起源，同时站在理论、模型的⻆度 讲解 Java 并发，
        
            让你的知识更成体系，融会贯通。
        
            最终让你能够得⼼应⼿地解决各种并发难题，同时将这些知识⽤于其他编程语⾔，让你的⼀分⾟劳三分收获。

```
